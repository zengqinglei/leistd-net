# 输出示例

## 示例 2 的输出

**输入**：
```
做个定时发布功能，用户可以设置发布时间，到点自动发布到小红书，失败要飞书通知我
```

**输出**：
```markdown
# REQ-20260606-XXX - 定时发布功能

## 需求描述
做个定时发布功能，用户可以设置发布时间，到点自动发布到小红书，失败要飞书通知我

## 验收标准

### Must have
- [ ] 用户可以设置发布时间（精度：分钟）
- [ ] 系统到点自动执行发布
- [ ] 发布成功/失败状态记录
- [ ] 发布失败时飞书通知用户

### Nice to have
- [ ] 发布前预览功能
- [ ] 批量定时发布
- [ ] 支持小红书多账号切换

## 风险评估
- **风险**: 小红书 API 权限问题
  **应对**: 提前申请企业账号授权
- **风险**: 定时任务调度可靠性
  **应对**: 使用消息队列（如 BullMQ）保证任务执行
- **风险**: 发布失败后的重试机制
  **应对**: 实现指数退避重试策略

## 涉及模块
- **内容管理模块**:
  - 定时任务列表 UI
  - 发布时间设置表单
- **平台账号模块**:
  - 发布 API 集成
  - 账号授权管理
- **通知模块**:
  - 飞书通知集成
  - 通知模板管理

---

*生成时间*: 2026-06-06T14:50:00.000Z
```
