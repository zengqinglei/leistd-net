# 降级策略 (Fallback Strategy)

> 当项目缺少规范文档时，Skill 的降级行为和提示机制

---

## 🎯 核心原则

### 1. 优雅降级

- ✅ **有规范**：按项目规范执行
- ✅ **无规范**：使用通用默认值 + **提示用户**
- ❌ **不报错停止**：继续执行，但告知用户

### 2. 提示而非阻塞

- ✅ **提示一次**：首次检测到缺失时提示
- ✅ **继续执行**：使用默认值完成当前任务
- ✅ **提供指引**：告诉用户如何创建规范文档

---

## 📋 降级场景

### 场景 1：无任务登记册 (registry.md)

**检测**：
```markdown
尝试读取：
1. docs/requirements/registry.md
2. docs/registry.md

结果：都不存在
```

**降级行为**：
```markdown
1. 查询任务进度：
   - 提示：⚠️ 未找到需求登记册。
   - 建议：建议创建 docs/requirements/registry.md 以跟踪任务进度。
   - 替代：从任务上下文文件汇总进度

2. 创建新任务：
   - 继续执行：初始化任务上下文
   - 自动创建：创建 registry.md 并登记
   - 提示：已为您初始化需求登记册
```

---

### 场景 2：无任务上下文文件

**检测**：
```markdown
尝试读取：
~/.openclaw/agents/{agentId}/task-context/{taskId}.md

结果：不存在
```

**降级行为**：
```markdown
1. 查询任务进度：
   - 提示：⚠️ 未找到任务 {taskId} 的上下文。
   - 替代：尝试从 Plan.md 恢复基本信息

2. 更新任务进度：
   - 自动创建：初始化任务上下文文件
   - 从 Plan.md 恢复：填充基本信息
   - 提示：已为您初始化任务上下文
```

---

## 📊 降级提示模板

### 任务登记册缺失

```markdown
⚠️ 未找到需求登记册 (docs/requirements/registry.md)。

建议创建需求登记册以跟踪任务进度。

或告诉我您的新需求，我可以帮您创建 Plan.md 并初始化登记册。
```

### 任务上下文缺失

```markdown
⚠️ 未找到任务 {taskId} 的上下文。

已尝试从 Plan.md 恢复基本信息。
建议定期保存任务上下文以跟踪进度。
```

## ✅ 最佳实践

### 1. 首次提示时机

- ✅ **首次检测到缺失时**：立即提示
- ✅ **只提示一次**：避免重复打扰

### 2. 提示方式

- ✅ **友好语气**：建议而非命令
- ✅ **明确路径**：告诉用户具体文件路径

### 3. 降级执行

- ✅ **使用默认值**：确保任务能继续
- ✅ **记录日志**：在任务上下文中记录降级行为
