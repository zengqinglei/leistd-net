# 部署工作流参考

> 仅在执行 `deploy`、`rollback` 或需要生成部署报告时读取。

## 预检查

- 工作区状态符合项目发布规范。
- Phase 4 测试报告通过。
- `code-review` 无 P0 问题。
- Plan.md 的 Must have 已验证或明确标记未覆盖。
- 目标环境、版本、服务器、命令和回滚方案已获得用户确认。

## 项目配置读取

1. 优先读取 `docs/standards/agent-workflow.md` 的部署索引。
2. 按索引读取 `docs/deploy/` 相关文档。
3. 常见配置：`docker-compose.yml`、`compose.yml`、`docs/deploy/server-config.md`、`.env.deploy`。
4. 缺少配置时，只准备方案；不要猜测生产服务器。

## 执行顺序

1. 备份配置。
2. 构建或拉取镜像。
3. 标记版本，优先使用 commit tag。
4. 更新配置。
5. 重启服务。
6. 健康检查、日志检查、核心功能验证。
7. 输出部署报告。

## 回滚

触发条件：健康检查失败、核心功能不可用、用户要求回滚、关键日志异常。

回滚前确认：目标环境、回滚版本、数据影响、回滚命令。

## 报告模板

```markdown
## 部署报告

- 结果：成功 / 失败 / 已回滚
- 环境：staging / production
- 版本：commit/tag
- 时间：YYYY-MM-DD HH:mm
- 健康检查：通过 / 失败
- 日志检查：通过 / 失败
- 访问地址：URL

## 验收闭环对照

| 验收项 | 状态 | 验证方式 | 证据 |
|--------|------|----------|------|
| xxx | ✅/⚠️/❌ | 健康检查/页面验证/API 验证 | URL/日志/截图 |
```
