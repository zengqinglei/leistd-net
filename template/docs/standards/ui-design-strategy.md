# UI/UX 设计策略

## 1. 设计原则

- 以用户任务为中心，而不是以数据表单为中心。
- 优先让 AI 辅助理解、生成、补全和校验，保留表单作为精确控制与兜底。
- 信息层级清晰：主要动作、关键状态、辅助信息分层呈现。
- 错误态、空状态、加载态必须可解释并可恢复。

## 2. AI 协作界面原则

- AI 建议必须可追溯来源或说明推理依据。
- 关键操作前提供确认，不自动执行高风险动作。
- 用户可以编辑、撤销或拒绝 AI 生成内容。
- 对不确定结果明确展示置信度、限制或待确认项。

## 3. 组件规范

- 组件职责单一，避免将业务流程、数据请求和展示状态混在一个文件中。
- 表单字段应有标签、帮助文本、校验提示和提交状态。
- 列表应考虑空态、筛选、分页、加载和错误重试。
- 对话式组件应支持历史、重试、中断、复制和反馈。

## 4. 可访问性

- 交互元素可通过键盘访问。
- 颜色不作为唯一信息表达方式。
- 表单错误与字段关联。
- 动画不影响主要操作。

## 5. 文档要求

涉及 UI 的需求 Plan 应说明：

- 目标用户与使用场景。
- 页面/组件结构。
- 状态流转。
- 关键交互和异常态。
- 验收与截图/录屏要求。
